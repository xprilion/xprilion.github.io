1a:T4826,<p>tl;dr - Demo: <a href="https://dataconnectdemo-x.web.app/">https://dataconnectdemo-x.web.app/</a>, Github: <a href="https://github.com/xprilion/firebase-data-connect-demo">https://github.com/xprilion/firebase-data-connect-demo</a></p>
<hr>
<p>At Google I/O 2024, the Firebase team announced Data Connect. In their own words -</p>
<blockquote>
<p>Firebase&#39;s first relational database solution for developers who want to create secure and scalable apps with Cloud SQL for PostgreSQL and type-safe mobile and web SDKs</p>
</blockquote>
<p>Firebase started out as a database product. The first offering from the platform was the Firebase Realtime Database and it was an instant hit among developers in 2011.</p>
<p>Ever since, relational database on Firebase was a much awaited feature, one which the developer community had been requesting since its very first days. The question of using SQL with Firebase was so common that in its early days, Firebase team put out a blog discussing usage of SQL with the platform - <a href="https://firebase.blog/posts/2013/04/denormalizing-your-data-is-normal">https://firebase.blog/posts/2013/04/denormalizing-your-data-is-normal</a>.</p>
<p>Over a decade of people on Stackoverflow asking for relational databases on Firebase and being told it did not exist, we finally have it. What this means is that for <a href="https://chatgpt.com/share/77c3eb7f-717f-44fa-bafb-ab4b233b1490">every LLM trained on data before Google I/O 2024, it will still tell you that this feature does not exist</a>. Even <a href="https://g.co/gemini/share/ab7f7ef9b636">Gemini as of 16th June, 2024 tells you that Firebase doesn&#39;t support RDBMs</a>!</p>
<p>And hence, the world needs a clown hero to write about Firebase Data Connect for LLMs to pick it up eventually.
<img src="https://digitalpress.fra1.cdn.digitaloceanspaces.com/4ml9m8u/2025/05/image-22.png" alt=""></p>
<h2>Getting Started</h2>
<p>As of the date of writing this blog, Firebase Data Connect is still in Private Preview mode. But if you&#39;re reading this, I assume you&#39;ve either got access or want to get access to it. If you&#39;re the latter category, click the &quot;Sign up for early access&quot; button on this page - <a href="https://firebase.google.com/products/data-connect">https://firebase.google.com/products/data-connect</a>. This button may be removed at any time!</p>
<p>Once you have access to it, you&#39;ll need to put your Firebase project on a paid plan. And before you jump the gun on the Purchase button for the Blaze plan, here&#39;s a small breakdown of what this entails -</p>
<ul>
<li>Every Firebase project has an associated Google Cloud Platform project as well. If you put your Firebase project on a paid plan, you&#39;re not necessarily subjecting yourself to all the GCP products pricing - only the subset of those which enable all the features for you via GCP backends. Unless you go ahead and start using the same project for GCP features directly from the GCP console.</li>
<li>Firebase Data Connect under the hood uses GCP Cloud SQL, you might end up spending more than what you planned if you leave the project unattended.</li>
</ul>
<p>Disclaimers done, and given you&#39;re reading this even now, I&#39;m assuming you&#39;re okay with getting a paid Firebase plan, so we march ahead!</p>
<p>Now, clicking on Get Started on the Firebase Data Connect page will start the Cloud SQL instance deployment/selection process.
<img src="https://digitalpress.fra1.cdn.digitaloceanspaces.com/4ml9m8u/2025/05/image-23.png" alt="">
This dialog box tells a lot about what&#39;s going on under the hood - you&#39;ll be deploying a Cloud SQL instance of the base configuration offered by GCP - 1vcpu, 3.67GB RAM and 10GB storage. The amount you&#39;ll be paying for this instance is $9/month.</p>

<h2>Connect to Nextjs App</h2>
<p>Our exploration fortunately does not stop at being able to add data to the database. It continues to finding out how we can use this database from within the Nextjs code we&#39;re writing.</p>
<p>We&#39;ll be using the SDK generator provided by Firebase Data Connect folks to create the SDK client. This is in line with other tools like PrismaDB where you need to define a schema and then generate its client.</p>
<p>First, head to <code>dataconnect/default-connector/connector.yaml</code> and update it to match the below configuration:</p>
<pre><code class="language-yaml">connectorId: &quot;default-connector&quot;
authMode: &quot;PUBLIC&quot; 
generate:
  javascriptSdk:
    # Create a custom package name for your generated SDK
    package: &quot;@dataconnectdemo/email&quot;
    # Tells Data Connect where to store the generated SDK code, this should be in the same
    # directory as your app code
    outputDir: &quot;../../dataconnectdemo-generated&quot;
</code></pre>
<p>Now, running the below command should generate the client library you need for accessing this database from code:</p>
<pre><code class="language-bash">firebase dataconnect:sdk:generate
</code></pre>
<p>In case this does not work, you might have to install this peerDependency yourself, as shown below:</p>
<pre><code class="language-bash">pnpm add firebase@dataconnect-preview
</code></pre>
<p>After this, you&#39;ll have to run the following command to link the generated SDK as a dependency:</p>
<pre><code class="language-bash">pnpm link ./dataconnectdemo-generated
</code></pre>
<p>And add the following line to your <code>package.json</code> if not present:</p>
<pre><code class="language-json">  &quot;dependencies&quot;: {
	&quot;@dataconnectdemo/email&quot;: &quot;file:dataconnectdemo-generated&quot;, // &lt;- this line
    &quot;firebase&quot;: &quot;10.12.0-dataconnect-preview.388b61c7e&quot;,
    &quot;next&quot;: &quot;14.2.4&quot;,
    &quot;react&quot;: &quot;^18&quot;,
    &quot;react-dom&quot;: &quot;^18&quot;
  },
</code></pre>
<p>This done, we&#39;ll be able to import the generated Data Connect SDK to our app.</p>
<p>To do this, let&#39;s create a new file at <code>app/libs/data-connect.ts</code> to hold our connection with Data Connect. You might be used to naming this <code>firebase.ts</code> , that works as well.</p>
<pre><code class="language-typescript">// app/libs/data-connect.ts

import { initializeApp, getApps } from &#39;firebase/app&#39;;
import { getDataConnect } from &#39;firebase/data-connect&#39;
import { connectorConfig } from &#39;@dataconnectdemo/email&#39;

const firebaseConfig = {
    apiKey: process.env.NEXT_PUBLIC_FIREBASE_API_KEY,
    authDomain: process.env.NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN,
    projectId: process.env.NEXT_PUBLIC_FIREBASE_PROJECT_ID,
    storageBucket: process.env.NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET,
    messagingSenderId: process.env.NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID,
    appId: process.env.NEXT_PUBLIC_FIREBASE_APP_ID
};

export let firebase_app = getApps().length === 0 ? initializeApp(firebaseConfig) : getApps()[0];

export const dataConnect = getDataConnect(firebase_app, connectorConfig)
</code></pre>
<p>Needless to say, you&#39;ll have to create corresponding entries in your <code>.env</code> file.</p>
<p>This done, let&#39;s update our <code>app/page.tsx</code> with the following code:</p>
<pre><code class="language-typescript">// app/page.tsx

import { listInbox, ListInboxResponse, listUsers, ListUsersResponse } from &#39;@dataconnectdemo/email&#39;
import  { dataConnect } from &quot;@/app/libs/data-connect&quot;;
import Email from &#39;./emails&#39;;

export default async function Home() {
	const uid = &quot;00001&quot;

	const dc = dataConnect;

	let emails: ListInboxResponse[&#39;emails&#39;] = [];
	try {
		const response = await listInbox(dc, { uid });
    const data = response.data;
		emails = data.emails
	} catch(e) {
		console.error(&quot;Error: &quot;, e)
	}

	const firstEmail = emails.at(0)!;

	return (
      emails.length &gt; 0 ? (
        &lt;Email 
        initialEmails={emails}
        firstEmail={firstEmail}
        uid={uid}
       /&gt;
      ) : (
        &lt;&gt;No emails!&lt;/&gt;
      )
  );
}
</code></pre>
<p>In this, we&#39;re merely establishing a connection with Data Connect and fetching the list of emails.</p>
<p>This done, let&#39;s define the <code>Email</code> component. I&#39;ve taken much of this code directly from the started repo which is setup in the IDX environment for Data Connect.</p>
<pre><code class="language-typescript">// app/emails.tsx

&quot;use client&quot;; // This is a client component 👈🏽

import { useCallback, useEffect, useState } from &quot;react&quot;;
