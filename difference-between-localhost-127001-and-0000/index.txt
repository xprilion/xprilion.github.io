16:T7d1,<p>I've been often asked, why do we sometimes need to specify the IP for a program to be <code>0.0.0.0</code> in order to get it to be accessible from machines outsides the network. I shall attempt to answer the question in this post.</p>
<h2>What is 127.0.0.1</h2>
<p><code>127.0.0.1</code> is the loopback address. It means, whatever request is made, send it back to the same machine. When a remote host calls this, they're actually calling themselves.</p>
<h2>What is <a href="http://localhost" rel="noopener noreferrer">localhost</a></h2>
<p><a href="http://localhost" rel="noopener noreferrer"><code>localhost</code></a> is simply a conventional mapping done in the <code>hosts</code> file of the OS. It can be changed. I could ask <a href="http://google.com" rel="noopener noreferrer"><code>google.com</code></a> to point to my loopback address and that would work same.</p>
<h2>What is 0.0.0.0</h2>
<p><code>0.0.0.0</code> is a wildcard for all open routes to the system within the system where the script runs.</p>
<h2>When do you face the need to differentiate between them -</h2>
<p>We specifically face the problem of differentiating them in Flask/Django/NodeJS applications because these are runtimes which require the user to define which IP (or routes) to listen on. If you've ever tried web development on PHP, you'd realize that the <a href="http://localhost" rel="noopener noreferrer"><code>localhost</code></a> web server was accessible using the IP of the computer's LAN as well from other devices (say your mobile). This is because by default Apache listens on <code>*:80</code> which essentially means <code>0.0.0.0:80</code>, as given in its default virtual host configuration file -</p>
<pre><code class="language-markdown"># Ensure that Apache listens on port 80
Listen 80
&lt;VirtualHost *:80&gt;  # Notice this
    DocumentRoot "/var/www/html" # OR "C:/xampp/htdocs" for Windows systems with XAMPP installation

    # Other directives here
&lt;/VirtualHost&gt;
</code></pre>
