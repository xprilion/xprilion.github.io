# Hello, WebMCP

## Overview

Duration: 1

Hey there! This codelab introduces you to WebMCP - the very early W3C  draft submitted by Google and Microsoft to enable better AI agents in the browser. In the shortest, and as per the draft report: **the WebMCP API enables web applications to provide JavaScript-based tools to AI agents.**

You can read the draft report here: [https://webmachinelearning.github.io/webmcp/](https://webmachinelearning.github.io/webmcp/)

## Requirements

Duration: 3

In order to follow this codelab, you'll need the following:
- Python on local machine
- Chrome: Version 146.0.7672.0 or higher.
- Gemini API Key, obtain a free one here - https://ai.dev/apikey

Enable WebMCP on your browser:
- Open Chrome and navigate to chrome://flags/#enable-webmcp-testing
- Set the flag to Enabled.
- Relaunch Chrome to apply the changes.

Additionally, you can install this extension by Google to test out WebMCP:
[WebMCP - Model Context Tool Inspector - Chrome Web StoreInspect, monitor, and execute WebMCP tools manually or with Gemini![](https://digitalpress.fra1.cdn.digitaloceanspaces.com/4ml9m8u/icon/icon_144px.png)Chrome Web Store![](https://digitalpress.fra1.cdn.digitaloceanspaces.com/4ml9m8u/thumbnail/dwvaKnrF0HWleaF2JPEEEOBF5RJYsRislvt7HePRXJnlPL65lU2QYdUAsOej31bqgrZSFyDcRUnKlSLv_36IZzgEEQ-s128-rj-sc0x00ffffff)](https://chromewebstore.google.com/detail/webmcp-model-context-tool/gbpdfapgefenggkahomfgkhfehlcenpd)
This codelab assumes you're on a Linux environment (Mac/Windows WSL/linux distros, etc.). If you're on Windows or any other operating system, you may have to substitute some of the commands to make them work.

## Setup Repo

Duration: 3

Let's setup our repository for the day.

We'll be working with this repository - [https://github.com/xprilion/webmcp-train-your-baby-dragon](https://github.com/xprilion/webmcp-train-your-baby-dragon)

Clone it using:

```bash
git@github.com:xprilion/webmcp-train-your-baby-dragon.git
```

Switch to `main` branch

```bash
git checkout main
```

Assuming you've Python on your device,

```bash
python -m http.server
```

With this done, you should be able to navigate to `http://localhost:8000` to see the web page rendered.
