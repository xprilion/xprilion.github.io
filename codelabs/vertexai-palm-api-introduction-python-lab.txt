15:T2185,<h2>Overview</h2>
<p>Duration: 1</p>
<p>Hey there!</p>
<p>In this Codelab, we&#39;ll walk through creating a simple chatbot based game using Google Cloud&#39;s Vertex AI and Flask, a popular web framework in Python. Our application will ask you to make a guess about what the bot is thinking and the bot will respond with hints about what it is thinking and how far away your guess is from the correct item.</p>
<h2>Requirements</h2>
<p>Duration: 2</p>
<p>In order to follow this codelab, you&#39;ll need the following:</p>
<ul>
<li>A Google Cloud Platform account with active billing.</li>
<li>A development environment with Python 3.7 or above installed.</li>
<li>Access to terminal/shell for executing commands.</li>
</ul>
<h2>Setting Up Your Google Cloud Environment</h2>
<p>Duration: 10</p>
<ul>
<li>Log in to your <a href="https://console.cloud.google.com/">Google Cloud Platform</a> account.</li>
<li>Create a new project, let&#39;s name it <code>project-x</code>.</li>
<li>Enable <a href="https://cloud.google.com/billing/docs/how-to/modify-project">billing for the project</a>.</li>
<li>Navigate to the <a href="https://console.cloud.google.com/apis/dashboard">APIs &amp; Services dashboard</a> and enable the Vertex AI API.</li>
<li><a href="https://cloud.google.com/iam/docs/keys-create-delete">Create a service account and download the JSON key file</a> (<code>key.json</code>). Store this file safely.</li>
</ul>
<h2>Setting Up Your Local Environment</h2>
<p>Duration: 5</p>
<p>Install other required libraries:</p>
<pre><code class="language-bash">pip install marko google-cloud-aiplatform
</code></pre>
<p>Install Flask:</p>
<pre><code class="language-bash">pip install Flask
</code></pre>
<h2>Write the driving code</h2>
<p>Duration: 10</p>
<ul>
<li><strong>Importing Libraries</strong>: We start by importing necessary Python libraries. Flask for our web framework, Vertex AI for AI model interaction, and others for various functionalities.</li>
</ul>
<pre><code class="language-python">import os
from flask import Flask, request, Response, g, render_template, jsonify
import marko
import vertexai
from vertexai.language_models import TextGenerationModel
</code></pre>
<ul>
<li><strong>Initializing the App and Vertex AI</strong>: We initialize our Flask app and configure Vertex AI with our GCP project details.</li>
</ul>
<pre><code class="language-python">
app = Flask(__name__)
app.debug = True

vertexai.init(project=&quot;project-x&quot;, location=&quot;us-central1&quot;)
parameters = {
    &quot;temperature&quot;: 1,
    &quot;max_output_tokens&quot;: 256,
    &quot;top_p&quot;: 0.8,
    &quot;top_k&quot;: 40
}

model = TextGenerationModel.from_pretrained(&quot;text-bison@001&quot;)
</code></pre>
<ul>
<li><strong>Defining Routes</strong>: We define two routes - one for the home page and another for handling chat messages.</li>
</ul>
<pre><code class="language-python">@app.route(&#39;/&#39;, methods=[&#39;GET&#39;])
def hello_world():
    return render_template(&quot;chat.html&quot;)

@app.route(&#39;/chat/&lt;guess&gt;/&lt;actual&gt;&#39;, methods=[&#39;GET&#39;])
def chat(guess, actual):
    response = model.predict(
        &quot;You are the bot in a guessing game where the player tries to guess a secret item you are thinking about. &quot; +
        &quot;The rules of the game are: 1. the player will make a guess. 2. if the guess is correct or very close to the correct answer, you will say, &#39;Congratulations, you&#39;ve got it right!&#39; &quot; +
         &quot;3. If they are not very close to the actual item, respond with a humorous remark about their guess. Then without mentioning the actual item, &quot; + actual + &quot;, provide a subtle hint to guide the player closer to the secret item.&quot; +
        &quot;The player has just guessed &quot; + guess + &quot;.&quot;,
        **parameters
    )

    return jsonify({
        &quot;response&quot;: marko.convert(response.text)
    })

if __name__ == &#39;__main__&#39;:
    app.run(debug=True, host=&quot;0.0.0.0&quot;, port=int(os.environ.get(&quot;PORT&quot;, 8080)))
</code></pre>
<p>With the backend done, we&#39;re free to implement the UI for the API in any manner we want to.</p>
